# 小型CMS审计思路

```r
旧的框架漏洞
如果是基于某个框架的，对应版本看一下框架的漏洞是否在该CMS中存在，存在就比较舒服了

SQL注入漏洞
数据库操作相关，跟着输入找和数据库操作相关的部分，如果是基于框架的，由于框架基本上已经封装好了常用的操作，不太可能有SQL注入的漏洞；如果是自己写的，那就很有可能存在SQL注入漏洞

XSS漏洞
所有可输入的地方都试一下

上传漏洞
上传文件/头像等没有做好过滤，可能上传代码文件，获取webshell

删除漏洞
删除的逻辑没有做好，可能导致任意文件删除的漏洞

任意文件读取
读取文件相关的操作，没有做好限制，导致可以读取任意文件

权限提升漏洞
比如从普通用户提升权限为管理员

CSRF添加超级管理员漏洞
可以在系统唯一的超级管理员上再加一个超级管理员


现在就知道这些了，审计CMS最大的好处就是你知道所有的代码实现，还可以通过调试去掌握代码逻辑，大部分不需要去猜
```
