# IDA Pro MCP

用MCP方式提供IDA功能，配合OpenCode挺好用的

github地址: https://github.com/mrexodia/ida-pro-mcp

安装
```
pip uninstall ida-pro-mcp
pip install https://github.com/mrexodia/ida-pro-mcp/archive/refs/heads/main.zip
```

修改智能体配置，添加ida-pro-mcp工具
```
ida-pro-mcp --install
```

提示词
```
你的任务是创建一份完整且全面的逆向工程分析报告。

请遵循以下系统化的分析方法：

1. **反编译分析**
   - 彻底检查反编译器的输出结果。
   - 添加详细的注释来记录你的发现。
   - 重点在于理解每个组件的实际功能和用途（不要依赖那些过时的、不正确的旧注释）。

2. **提升数据库中的代码可读性**
   - 将变量重命名为合理且具有描述性的名称。
   - 在必要时修正变量和参数的数据类型（特别是指针和数组类型）。
   - 更新函数名，使其能够准确反映函数的实际功能。

3. **按需深入分析**
   - 如果需要更多细节，请查看反汇编代码并添加包含发现的注释。
   - 记录那些仅靠反编译无法看清的底层行为。
   - 使用子代理（sub-agents）来执行详细分析。

4. **重要约束条件**
   - **绝对不要**自行进行进制转换——如有需要，请使用 `int_convert` MCP 工具。
   - 根据需要，使用 MCP 工具来获取信息。
   - 所有的结论都必须基于实际的分析得出，绝不能凭空假设。

5. **文档编写**
   - 生成全面的 `RE/*.md` 文件来记录你的发现。
   - 记录所采取的步骤和使用的方法论。
   - 当用户提出要求时，确保新内容的准确性高于之前的分析文件。

用中文写注释和文档
```
